# 1GHZ — 10.0.0.x

Windows XP SP3, 383 MB RAM, single 586-class CPU (~1 GHz), 228 GB C:.

## Auto-login

Configured via classic Winlogon autologon registry values under
`HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon`:

| Value | Data |
|---|---|
| `AutoAdminLogon` | `1` |
| `DefaultUserName` | `Administrator` |
| `DefaultDomainName` | `1GHZ` (local machine) |
| `DefaultPassword` | `CHANGE-ME` (REG_SZ, plaintext) |
| `ForceAutoLogon` | not set |

The password is stored in cleartext (no LSA secret). Anyone with read
access to that key can recover it. To reproduce on another machine:

```cmd
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v AutoAdminLogon /t REG_SZ /d 1 /f
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultUserName /t REG_SZ /d Administrator /f
reg add "HKLM\SOFTWARE\Microsoft\Windows NT\CurrentVersion\Winlogon" /v DefaultPassword /t REG_SZ /d CHANGE-ME /f
```

The local `Administrator` account password is `CHANGE-ME` to match.

## File share (Z:)

`Z:` is a persistent drive mapping to `\\10.0.0.x\Files`, stored in
`HKCU\Network\Z` (reconnected by Explorer at logon):

```
RemotePath     \\10.0.0.x\Files
ProviderName   Microsoft Windows Network
ConnectionType 1
```

No credentials are stored in the mapping itself and there is nothing in
Stored User Names and Passwords. The mount succeeds because the local
logged-in user is `Administrator` / `CHANGE-ME`, and the SMB server at
`10.0.0.x` accepts those credentials (its share account is
`admin` / `CHANGE-ME` — same password, and the server treats the
incoming `Administrator` as an equivalent local user).

In other words, the "trick" is: **make the local autologon account's
username/password match a valid account on the SMB server, then just
use a persistent `net use` with no explicit credentials.** No cmdkey,
no startup script, no plaintext password in a .bat file.

To reproduce on another machine:

```cmd
net use Z: \\10.0.0.x\Files /persistent:yes
```

Run that once while logged in as the autologon user. It will reconnect
on every subsequent login.

## Notes

- `cmdkey.exe` is not present on this XP install (stock SP3), so
  credentials cannot be inspected/added that way — the
  username-match approach is what's actually in use here.
- `DefaultPassword` in cleartext is the standard XP autologon
  mechanism. Sysinternals `Autologon.exe` would store it as an LSA
  secret instead; this machine does not use that.
